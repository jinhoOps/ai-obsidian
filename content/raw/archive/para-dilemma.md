# 🧠 PARA 방법론 도입과 지식 복리의 충돌

현재 구축 중인 지식베이스 에 티아고 포르테의 PARA(Projects, Areas, Resources, Archives) 방법론을 추가로 덧씌워야 할지에 대한 고민을 정리합니다.

## ⚖️ 비교 및 분석

### 1. PARA: 실행과 관리 (Doing)
- **장점**: '현재 내가 무엇을 하고 있는가'에 집중하여 정보의 가용성을 극대화함.
- **우려**: 계층적 폴더 구조가 지식의 유연한 연결을 방해하고, 에이전트가 정보를 탐색할 때 '사일로(Silo)' 현상을 유발할 수 있음.

### 2. Zettelkasten: 연결과 증식 (Thinking)
- **장점**: 정보 간의 원자적 연결을 통해 새로운 통찰을 유도하고, 지식의 복리(Compounding) 효과를 창출함.
- **우려**: 명확한 실행 단위(프로젝트)가 부족할 경우, 지식이 '쌓이기만 하는' 상태에 머물 위험이 있음.

## 🛠️ 에이전트 관점의 제안 (Strategy)

에이전트에게 중요한 것은 **"이 정보가 어디에 들어있는가"**보다 **"이 정보가 다른 지식과 어떻게 연결되는가"**입니다. 따라서 PARA의 엄격한 폴더링보다는 다음과 같은 하이브리드 접근을 제안합니다.

1. **프로젝트(P)의 태그화**: 물리적 폴더 이동 대신 프론트매터의 `status: active-project`와 같은 메타데이터로 실행 단위를 식별합니다.
2. **영역(A)의 MOC(Map of Content)화**: 계층 구조 대신 위키의 인덱스 페이지(MOC)를 통해 영역을 유기적으로 구분합니다.
3. **자원(R)의 Wiki화**: 모든 지식은 위키의 원자적 노트로 존재하며, 특정 주제에 얽매이지 않고 필요할 때 인용/소환됩니다.
4. **보관(A)의 자동화**: 에이전트가 일정 기간 참조되지 않거나 완료된 정보의 우선순위를 낮추는 방식으로 '보관'의 목적을 달성합니다.

## 📌 결론
PARA의 **'실행 중심 필터링'** 철학은 수용하되, 지식의 연결성을 저해하는 **'물리적 폴더 구조'**는 지양하는 것이 본 시스템의 철학(지식 복리)에 가장 부합하는 방향입니다.
