# ⚖️ PARA vs Zettelkasten: 구조적 정합성 고찰

지식 관리 시스템의 두 가지 주요 방법론인 PARA(프로젝트 중심)와 Zettelkasten(연결 중심) 사이의 균형점을 탐구합니다.

## ⚖️ 방법론 비교

### PARA (Doing)
- **목적**: 현재 실행 중인 작업의 가용성 극대화.
- **구조**: Projects, Areas, Resources, Archives.
- **특징**: '지금 당장 무엇을 하는가'에 집중.

### Zettelkasten (Thinking)
- **목적**: 정보 간의 유기적 연결과 지식 증식.
- **구조**: 탈계층적 연결 구조.
- **특징**: '이 지식이 다른 지식과 어떻게 연결되는가'에 집중.

## 🛠️ 에이전트 중심의 하이브리드 전략 (Strategy)

에이전트가 탐색하기 가장 효율적인 정보 구조를 위해 다음과 같은 접근을 제안합니다.

1. **프로젝트(P)의 태그화**: 물리적 폴더 대신 프론트매터의 `status: active-project` 태그 활용.
2. **영역(A)의 MOC화**: 계층 구조 대신 위키의 인덱스 페이지(MOC)를 통한 영역 구분.
3. **자원(R)의 Wiki화**: 모든 지식은 위키의 원자적 노트로 존재하며 주제에 얽매이지 않고 인용됨.
4. **보관(A)의 자동화**: 참조 빈도가 낮은 정보의 우선순위를 에이전트가 자동으로 관리.

## 📌 결론
PARA의 **'실행 중심 필터링'** 철학은 수용하되, 지식의 연결성을 저해하는 **'물리적 폴더 구조'**는 지양하여 지식 복리 효과를 극대화합니다.
