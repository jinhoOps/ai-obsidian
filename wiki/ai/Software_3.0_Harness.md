# 🏗️ Software 3.0 시대의 하네스와 조직 생산성

LLM 도입이 개인의 센스에 맡겨지는 '각자도생'의 단계를 넘어, 조직 전체의 생산성 저점(Raising the Floor)을 높이기 위한 전략적 하네스(Harness) 구축이 필요합니다.

## 🚀 주요 개념

### 1. Executable SSOT (실행 가능한 단일 진실 공급원)
기존의 위키나 문서는 작성되는 순간부터 낡은 정보가 되지만, **플러그인 형태의 지식**은 사람이 읽는 매뉴얼이자 LLM이 즉각 수행하는 시스템 프롬프트가 됩니다. 문서의 패러다임이 '기록'에서 '실행'으로 전환되는 것입니다.

### 2. Raising the Floor (생산성 저점 상향 평준화)
도메인 맥락(Domain Context)이 담긴 AI 워크플로우를 플러그인화하여 팀원들에게 배포함으로써, 팀 내 LLM 활용 역량의 편차를 줄이고 전체 품질을 상향 평준화합니다.

### 3. Workflow Marketplace
조직 내에서 검증된 AI 워크플로우를 배포하고 피드백을 통해 고도화하는 플랫폼입니다. 이는 단순한 도구 확장을 넘어, 조직의 '일하는 방식'을 전파하는 거버넌스 도구로 기능합니다.

## 🏢 지식의 계층화 (Layered Architecture)

신입사원에게 모든 정보를 한꺼번에 주지 않듯, LLM에게도 작업에 필요한 지식만 계층화하여 주입해야 합니다.

1.  **조직/도메인 계층**: 전사적 규칙, 보안 가이드라인, Git 전략.
2.  **프로젝트 계층**: 해당 리포지토리의 아키텍처, 코드 컨벤션, 의존성 규칙.
3.  **작업/기능 계층**: 특정 기능을 구현하거나 버그를 고치기 위한 좁은 범위의 맥락.

## 🔄 데이터 플라이휠 (Data Flywheel)
하네스를 통해 규격화된 고품질의 작업 데이터가 축적되면, 이를 통해 도메인 특화 모델(sLLM)을 파인튜닝하고, 이를 다시 워크플로우의 평가 기준으로 활용하는 선순환 구조를 구축할 수 있습니다.
